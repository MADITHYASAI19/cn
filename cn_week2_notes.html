Topic 1: DNS — Domain Name System Very High Weightage

What is DNS?

DNS is a global distributed database for the internet that primarily translates human-readable domain names (like www.iitkgp.ac.in) into IP addresses (like 203.110.x.x). Routers understand only IP addresses — not names. DNS is the "phonebook" of the internet.

Key explanation points

1Why DNS is needed: IP addresses are hard to remember. Routers can only process IP addresses (at the network layer). Names are understood only at the application layer. DNS bridges this gap by resolving names to IPs before communication begins.
2Three categories of domains: (a) Generic TLD (gTLD) — 3+ characters: .com, .org, .net, .edu, .aero, .museum etc. Managed by ICANN. (b) Country TLD (ccTLD) — exactly 2 characters: .in (India), .us (USA), .au (Australia), .ca (Canada). (c) Inverse/Reverse domain — IP to name resolution, uses in-addr.arpa.
3Domain namespace hierarchy (tree structure): Root (.) → TLD (.in) → Second level (.ac) → Third level (iitkgp) → Subdomain (cse). Each node in this tree has a label. Reading right to left gives you the full path. Example: cse.iitkgp.ac.in — cse is leftmost, in is rightmost (TLD).
4FQDN — Fully Qualified Domain Name: A domain name ending with a dot (.) is fully qualified and complete. Example: cse.iitkgp.ac.in. (with trailing dot). Without trailing dot = partially qualified = incomplete. DNS resolver may append a suffix to complete a partially qualified name.
5Zone vs Domain: A Domain is the entire namespace under a label (e.g., iitkgp.ac.in and all below it). A Zone is a domain MINUS the sub-domains delegated to other DNS servers. A domain is served by two or more DNS servers for redundancy. Only ONE primary (master copy). Secondary servers sync via zone transfer.
6Zone transfer: Primary server loads data from disk. Secondary server loads from the primary. When primary downloads from secondary = zone transfer. Used for keeping secondary servers updated and synchronized.
7Resource Records (RR): DNS database entries. Structure: Name | Type | Class | TTL | RDLength | RData. Provides mapping between domain name and network objects. A zone starts with a Start of Authority (SOA) record. NS record identifies which server is authoritative.
8Recursive vs Iterative resolution: Recursive — DNS server itself queries other servers and returns final answer to client. Iterative — DNS server returns address of NEXT server to query if it doesn't know; client queries each server itself. Both are valid.
9Two types of resolvers: (a) Full resolver — maintains a cache, queries name servers on behalf of user. (b) Stub resolver — library routine linked INTO the user program (getHostByName, getHostByAddress). Faster, more popular, hits the name server directly.
10Inverse DNS (Reverse lookup): Normal = name → IP (forward). Inverse = IP → name (reverse). Used for authentication — e.g., FTP server receiving IP 153.2.79.9 wants to know which domain it belongs to. Uses in-addr.arpa. IP is written in reverse: 9.79.2.153.in-addr.arpa.

Common mistakes

  • Confusing zone and domain — zone = domain minus delegated sub-domains
  • Thinking DNS uses TCP — DNS primarily uses UDP (port 53). TCP used for zone transfers
  • Confusing recursive and iterative — in recursive, the server does all the work. In iterative, the client does multiple queries
  • In inverse DNS, IP is written in REVERSE order before appending in-addr.arpa
  • BIND (Berkeley Internet Name Domain) is the most popular DNS server software, runs as "named" daemon

Topic 2: Client-Server Model & FTP/TFTP Important

Client-server paradigm

The dominant model for application communication. Server = process offering a service, always waiting/listening. Client = process requesting the service. They may be on the same machine or different machines. The five-tuple {Protocol, Server IP, Server Port, Client IP, Client Port} uniquely identifies every connection.

Key explanation points

1Iterative server: Handles ONE client at a time, sequentially. Server knows in advance how long each request takes. Single copy of server process runs. Other clients WAIT. Used when requests are short and fast.
2Concurrent server: Handles MULTIPLE clients simultaneously. Server forks (creates) a child process for EACH client request. Server process immediately goes back to listening. Used when request duration is unknown. Most HTTP servers are concurrent.
3Socket: Mechanism for Inter-Process Communication (IPC). Like a file handle — you open a socket, get a socket ID, and use it for communication. Berkeley socket interface is standard. Analogy: a telephone — it allows one person to speak to another.
4Five-tuple for a connection: {Protocol, Client IP, Client Port, Server IP, Server Port}. ANY difference in any one field = different connection. This is why multiple browser tabs to same server work independently.
5FTP — File Transfer Protocol: RFC 959. Uses TCP (reliable). Uses TWO separate connections: Control connection (port 21) — sends commands. Data connection (port 20) — transfers actual files. Only ONE data transfer at a time per session.
6FTP active vs passive mode: Active mode — server initiates data connection to client (server uses port 20). Passive mode — client initiates data connection to server (server uses an ephemeral port > 1023). Passive is used when client is behind a firewall/NAT.
7FTP has two processes at each end: PI (Protocol Interpreter) — handles the control connection and commands. DTP (Data Transfer Process) — handles the actual data transfer. Both client and server have PI + DTP.
8File transfer modes in FTP: ASCII mode — for text files (.txt, .html). Binary mode — for non-text files (.doc, .pdf, .jpg, .mp4). You must explicitly set the mode before transfer.
9TFTP — Trivial FTP: Uses UDP (port 69) — NOT TCP. Connectionless, no authentication, no directory listing. Only: read a file from server OR write a file to server. Used for: router/device firmware updates at boot time, constrained-resource devices. Lower overhead than FTP.

Topic 3: HTTP & HTML Very High Weightage

HTTP — Hypertext Transfer Protocol

The foundation of the World Wide Web. HTTP allows web documents to be communicated over the network. Web browser = HTTP client. Web server = HTTP server. Well-known port: 80. HTTP 1.0 (RFC 1945) — non-persistent. HTTP 1.1 (RFC 2616) — persistent connections by default.

1Three types of web documents: (a) Static — fixed page, same for all users. Request → get same page. (b) Dynamic — server executes a program and generates HTML on the fly based on request. Uses CGI (Common Gateway Interface). (c) Active — program runs on CLIENT side. Java applets, JavaScript. Called "client-side scripting."
2URL — Uniform Resource Locator: Structure = protocol://hostname:port/path. Default port for HTTP = 80. If port 80, it is optional to specify. Example: http://www.iitkgp.ac.in:80/index.html is same as http://www.iitkgp.ac.in/index.html.
3HTTP request message format: Request line (method + URL + HTTP version) → Header lines → Blank line → Body (optional). The blank line MUST be present even if no body exists.
4HTTP response message format: Status line (HTTP version + status code + phrase) → Header lines → Blank line → Body (document).
5HTTP is stateless: Server does NOT remember previous requests. Each request is independent. Cookies are used to maintain state/session information across requests.
6HTTP proxy server: Intermediate server between client and HTTP server. Can cache responses, filter content, log access, improve performance. Sits transparently between client and server.
7Transport independence: HTTP is designed to work over any transport, but TCP is the default and predominant choice. HTTP itself does not mandate a specific transport layer.

HTML — Hypertext Markup Language

  • Language for creating web pages. Extension: .html or .htm
  • Tags tell browser how to DISPLAY content. Tags come in pairs: <tag> content </tag>
  • Tags are NOT case-sensitive (but lowercase is standard)
  • Structure: <html> → <head> (title, meta) + <body> (content) → </html>
  • Key tags: <h1>-<h6> headings, <p> paragraph, <a href> hyperlink, <img src> image, <div> section/division
  • Three types of links: same folder, different folder (relative), external URL (absolute)
  • HTML is parsed by browser's HTML interpreter/parser to render the page

Topic 4: Telnet Protocol Moderate

What is Telnet?

A protocol that allows remote login to another system over a network. It provides a general, bidirectional, 8-bit byte oriented communication facility over TCP. Port 23 is the well-known Telnet port. Data and control use the SAME TCP connection (unlike FTP which uses two).

1NVT — Network Virtual Terminal: An intermediate representation of a generic terminal. Provides a standard language for communicating terminal control functions. Without NVT, escape characters and control functions from different terminal types would conflict.
2Same connection for data and control: Unlike FTP (separate ports 20 and 21), Telnet uses ONE single TCP connection for BOTH commands and data. This is a key distinguishing fact for NPTEL MCQs.
3IAC code (Interpret As Command): Special escape character, value = 255 (decimal). Any byte with value 255 in the stream is interpreted as a command prefix. If actual data is 255, it must be followed by ANOTHER 255. IAC followed by any other code = Telnet command.
4Negotiated options: Both endpoints negotiate mutually acceptable options — line mode vs character mode, echo mode, character set (ASCII vs EBCDIC). Options are NOT part of the Telnet protocol itself, allowing new features without changing the protocol.
5Control functions: IP (Interrupt Process), AO (Abort Output), AYT (Are You There — checks if system alive), EC (Erase Character — delete last char), EL (Erase Line — delete current line).
6Telnet as a generic TCP client: Telnet can be used to test ANY TCP server that uses ASCII protocols. Example: Telnet to port 80 to test HTTP server. Telnet to port 25 to test SMTP server. Sends whatever you type, prints whatever comes back.
7Default Unix/Linux servers accessible via Telnet: Echo server (port 7), Discard server (port 9), Daytime server (port 13), Chargen server (port 19).

Key comparison: Telnet vs FTP vs HTTP

  • Telnet: ONE connection (data + control together), port 23, remote login
  • FTP: TWO connections (control port 21, data port 20), file transfer
  • HTTP: ONE connection, port 80, stateless, web documents

Topic 5: SMTP & SNMP High Weightage

SMTP — Simple Mail Transfer Protocol

Protocol for transferring email between mail servers and from client to server. Uses TCP, port 25. RFC 821 (1982). Goal: reliable transfer of mail from sender to receiver. The email system analogy is the postal system — MTA = postal worker.

1Two components of SMTP systems: User Agent (UA) — prepares the message, creates the envelope (like Thunderbird, Outlook). Mail Transfer Agent (MTA) — transfers the mail across the internet. MTA acts as both client and server.
2Email message structure: Envelope (sender + receiver addresses + routing info) → Header (From, To, Subject, Date fields) → Blank line → Body (actual message content).
3SMTP keywords (commands): HELO (sender identifies itself), MAIL FROM (sender email), RCPT TO (recipient email), DATA (start of message body), QUIT (end session). Additional: RSET (reset), VRFY (verify address), NOOP, TURN, EXPN, HELP.
4SMTP reply codes (3-digit): 2xx = success. 3xx = command accepted, more info needed. 4xx = command rejected but temporary error (retry later). 5xx = command rejected permanently (bad user/address). Key: 220 = service ready, 250 = OK, 221 = closing.
5MIME — Multipurpose Internet Mail Extensions: SMTP was designed for 7-bit ASCII only. MIME extends SMTP to handle non-ASCII data (images, audio, video, binary). MIME converts non-ASCII to NVT 7-bit ASCII for transmission. Types: text, application, image, audio, video, multipart.
6Mail access protocols: SMTP = push protocol (delivers to server). POP3 (Post Office Protocol v3) = simple, downloads all emails, can delete from server. IMAP4 (Internet Message Access Protocol v4) = more features, can check headers before downloading, search by string, create/delete/rename mailboxes on server, access from anywhere.

SNMP — Simple Network Management Protocol

  • Purpose: Manage all aspects of the network. Monitor health, forecast capacity, troubleshoot, alter network parameters.
  • Transport: UDP (NOT TCP). This is frequently tested!
  • RFC 1157 defines SNMP community components
  • Three versions: SNMPv1 (1990) — plain text community string (no encryption). SNMPv2c (1996) — attempted security fixes but incomplete. SNMPv3 (2002) — full security: integrity + authentication + privacy (confidentiality). CIA properties.
1SNMP components (RFC 1157): MIB (Management Information Base) — text file describing managed objects using ASN.1 syntax. SNMP Agent — software on a network device (router, host, printer) maintaining configuration/state database. SNMP Manager — application querying/modifying the agent database. SNMP Sub-agent — subset of agent functions.
2SNMP operations: Get-Request (read one or more objects). Get-Next-Request (read next object in lexicographic order of OID). Set-Request (modify one or more objects). Get-Response (agent response to get/set requests). Trap (agent-initiated notification to manager without any query — triggered by specific events). Manager uses port 161. Traps go to port 162.
3OID — Object Identifier: Each managed object has a unique OID — a sequence of integers separated by dots, representable as a text string. MIB-2 (RFC 1213) defines managed objects for TCP/IP networks. Linux MIB files stored in a specific directory.
4SNMPv3 security features: Integrity — packet has not been tampered with. Authentication — message is valid/from authorized source. Privacy/Confidentiality — message cannot be read by unauthorized parties. These are the CIA properties of information security.

Tricky NPTEL MCQ facts — don't get these wrong

1SNMP uses UDP — NOT TCP (very commonly tested)
2Telnet uses ONE connection for both data and control — FTP uses TWO
3DNS uses UDP (port 53) for queries; TCP for zone transfers (large data)
4TFTP uses UDP (port 69) — NOT TCP like FTP
5HTTP is stateless — does NOT remember previous requests
6HTTP 1.1 = persistent connections (default). HTTP 1.0 = non-persistent
7MIME converts non-ASCII to 7-bit NVT ASCII for SMTP transmission
8SMTP Trap goes to port 162 (not 161). Get/Set/Response = port 161
9In SMTP: MAIL FROM = sender. RCPT TO = recipient. DATA = body start
10FTP active = server initiates data connection. Passive = client initiates

Exam strategy for Week 2

  • DNS questions: Focus on types (forward/reverse), zone vs domain, recursive vs iterative, RR types
  • FTP questions: Always remember TWO connections. Active vs passive mode. RETR=download, STOR=upload
  • HTTP questions: Status codes (404, 200, 301, 500 are favorites). HTTP methods (GET, POST, HEAD)
  • SMTP questions: Port 25, TCP, MIME for non-ASCII, POP3 vs IMAP4 differences
  • SNMP questions: UDP, port 161/162, Trap=agent-initiated, MIB, SNMPv3 security
  • If asked which uses UDP: DNS, TFTP, SNMP. Everything else usually TCP
