Topic 1: Introduction to Computer Networks Exam Favorite

What is a Computer Network?

A collection of interconnected devices (computers, servers, routers) that can communicate and share resources. Think of it like a highway system — different vehicles (data packets) travel between cities (computers) using roads (network links) and intersections (routers).

Key Explanation Points

1Protocol: A set of rules governing communication between devices. Like traffic rules — everyone must follow them for orderly movement. Formally: "a controlled sequence of messages exchanged between two or more systems to accomplish a given task."
2Network Architecture: A blueprint showing how computers in a network communicate. It defines the physical components, functional organization, and protocols. Like the blueprint of a building — defines how rooms connect.
3Functionalities vs Protocols: Functionalities = what you want to achieve (e.g., file transfer). Protocols = rules you follow to achieve it (e.g., FTP). Architecture ties them together.
4Physical Layer (Layer 1): Handles actual signal transmission. Converts digital data ↔ analog signals. Devices: Hub, Repeater. All nodes in same collision domain AND same broadcast domain.
5Data Link Layer (Layer 2): Divides collision domains but nodes remain in same broadcast domain. Devices: Switch, Bridge. Uses MAC (hardware) addresses. Improves bandwidth utilization.
6Network Layer (Layer 3): Divides BOTH collision and broadcast domains. Devices: Router, L3 Switch. Handles routing between different networks. Uses IP addresses.
7Transport Layer (Layer 4): Provides process-to-process communication. Sits above network layer. Handles error control, flow control, traffic management.
8Collision Domain vs Broadcast Domain: Collision domain = segment where collisions occur. Broadcast domain = segment where broadcasts reach. Hub does NOT separate either. Switch separates collision domains. Router separates BOTH.
Device Layer Collision Domain Broadcast Domain
Hub / Repeater Layer 1 One (shared) One (shared)
Bridge / Switch Layer 2 Divided per port One (shared)
Router Layer 3 Divided Divided

Common Mistakes Students Make

  • Confusing collision domain with broadcast domain
  • Thinking switches divide broadcast domains — they do NOT
  • Saying hub is a Layer 2 device — it is Layer 1
  • Forgetting that routers divide BOTH domains

Topic 3: OSI & TCP/IP Protocol Stack Most Important

The OSI Model — 7 Layers (Remember: "All People Seem To Need Data Processing")

OSI = Open System Interconnection. Defined by ISO. 7 layers from physical to application.

Layer 7ApplicationHTTP, FTP, SMTP, DNS, Telnet, SNMP
Layer 6PresentationData formatting, encryption, compression
Layer 5SessionSession establishment and maintenance
Layer 4TransportTCP, UDP, RTP — process-to-process
Layer 3NetworkIP (IPv4, IPv6), MPLS — routing
Layer 2Data LinkEthernet, Wi-Fi, Bluetooth, UMTS, LTE
Layer 1PhysicalCables, signals, bits, repeaters, hubs

TCP/IP Stack — 4 or 5 Layers

TCP/IP is the actual internet standard. 5-layer version: Application → Transport → Network (Internet) → Data Link → Physical. 4-layer version merges Data Link + Physical into "Network Interface."

OSI Layer TCP/IP Layer Key Protocols Data Unit
Application (7) Application HTTP, FTP, SMTP, DNS Message/Data
Presentation (6) SSL/TLS, JPEG, MPEG Message
Session (5) NetBIOS, PPTP Message
Transport (4) Transport TCP, UDP, RTP Segment
Network (3) Internet (IP) IPv4, IPv6, ICMP, ARP, IGMP Packet/Datagram
Data Link (2) Network Interface Ethernet, Wi-Fi, Frame Relay Frame
Physical (1) Cables, Hubs, Repeaters Bits

Encapsulation — The Onion Principle

Data from higher layers becomes the "payload" (cargo) for lower layers. Each layer adds its own header. Like packing a gift: the gift (data) goes inside a box (transport), the box goes in a bag (network), the bag is carried by a vehicle (physical). At the destination, each layer unwraps its own envelope.

  • Application data → Transport adds header → becomes Segment
  • Segment → Network adds IP header → becomes Packet
  • Packet → Data Link adds MAC header → becomes Frame
  • Frame → Physical converts to Bits → transmitted

Cross-Layer Protocols (NPTEL Frequently Asks)

  • DNS — between Application and Transport layers
  • SNMP — between Transport and Network layers
  • ARP, DHCP — between Network and Data Link layers
  • ICMP — works alongside IP at Network layer

Key Protocol Features (3 Elements)

  • Syntax — Data format, signal levels, structure of messages
  • Semantics — Control information, error handling, meaning of each field
  • Timing — Speed matching, sequencing, synchronization

Topic 4: Circuit Switching vs Packet Switching High Weightage

Switching Networks — The Basic Concept

When two devices are far apart and need to communicate, intermediate "switching nodes" route data from source to destination. Like a postal system — your letter passes through multiple post offices before reaching the destination.

Circuit Switching

1Definition: A dedicated communication path is established between source and destination for the ENTIRE duration of communication. Like booking a private taxi — the car is yours from start to finish.
2Three Phases: (a) Connection establishment / path setup → (b) Data transfer → (c) Disconnection / teardown. Resources are reserved in ALL three phases.
3Dedicated resources: The bandwidth is reserved for the connection even if no data is being sent. Channel is wasted during silence periods (e.g., pauses in phone call).
4Transparent connection: Once established, it behaves like a direct wire — no additional overhead during data transfer.
5Fixed data rate: Both ends MUST operate at the same data rate. No flexibility.
6Blocking vs Non-blocking: If all circuits are busy, new connections are BLOCKED. Non-blocking = enough resources for all simultaneous connections.
7Types of switching: Space Division Switching, Time Division Switching (TDM), TDM Bus, and combinations.
8Primary use: Traditional telephone networks (PSTN). Our landline phone calls use circuit switching.

Packet Switching

1Definition: Data is broken into small packets. Each packet travels independently through the network and may take different routes. Like sending multiple postcards — each one may take a different route to reach the same destination.
2Store and Forward: Each intermediate node receives the ENTIRE packet, stores it temporarily, then forwards it to the next node. This is the fundamental mechanism.
3Packet structure: Each packet = Data portion + Control information (routing/addressing info to find the destination).
4Typical size: 1000 octets (1000 × 8 bits = 8000 bits).
5No dedicated path: Bandwidth is dynamically allocated. Much more efficient use of network resources.
6Rate conversion possible: Nodes can buffer and synchronize different-speed links. Unlike circuit switching, both ends need NOT operate at the same data rate.
7Priority possible: Some packets can be given higher priority over others.

Two Types of Packet Switching

Feature Datagram Virtual Circuit
Path Each packet chooses its own route independently Pre-established route; all packets follow same path
Setup phase Not required Required (call request + call accept)
Packet header Contains full destination address Contains Virtual Circuit Identifier (VCI)
Ordering Packets may arrive out of order Packets arrive in order
Routing decision Made at each hop for each packet Made once during setup
Node failure Route can be changed dynamically All circuits through failed node are lost
Example Internet (IP) ATM, Frame Relay, X.25

Circuit vs Packet Switching — Comparison

Property Circuit Switching Packet Switching
Bandwidth Guaranteed, dedicated Dynamic, shared, not guaranteed
Efficiency Low (wasted if idle) High (shared bandwidth)
Setup time Required before data transfer No setup needed (datagram)
Data rate Fixed; both ends must match Variable; rate conversion possible
Delay Constant after setup Variable (queuing delay)
Congestion At setup time During data transfer
Best for Voice (real-time, continuous) Data (bursty, tolerates delay)
Example Traditional phone (PSTN) Internet, email, web browsing

Common Mistakes

  • Forgetting that circuit switching wastes bandwidth even during silence
  • Confusing datagram and virtual circuit packet switching
  • Thinking virtual circuit = circuit switching (it is still packet switching!)
  • VCI (Virtual Circuit Identifier) ≠ destination address — VCI is local to each link

Topic 5: Application Layer & Key Protocols Exam Favorite

What is the Application Layer?

The topmost layer — directly interfaces with the user. It's what you SEE and USE. When you open a browser, send an email, or transfer a file, you're using application layer protocols. The application layer is "provided by programs that use TCP/IP communication" — it is a user process cooperating with another process (usually on a different host).

DNS — Domain Name System (Very Important)

  • Purpose: Translates human-readable domain names to IP addresses (e.g., www.iitkgp.ac.in → 203.xx.xx.xx)
  • Why needed: Routers understand IP addresses, not names. DNS acts as the "phone book" of the internet.
  • TLD examples: .in (India), .us (US), .edu (education), .com (commercial), .net (network), .org (organization)
  • Hierarchy: Root → TLD (.ac) → Domain (iitkgp) → Sub-domain (www)
  • DNS is between Application and Transport layers — a cross-layer protocol
  • 200+ TLDs exist on the internet

Sockets — The Communication Endpoint

  • A socket = IP address + Port number + Protocol
  • Uniquely identifies a process on the network
  • Server: Always listening on a known port (e.g., HTTP on port 80)
  • Client: Connects to server's IP + port, gets an ephemeral (temporary) port
  • 5-tuple identifies a connection: {Protocol, Client IP, Client Port, Server IP, Server Port}

TCP vs UDP — When to Use Which

  • TCP: Connection-oriented, reliable, ordered, flow control, congestion control. Use for: HTTP, FTP, SMTP, Telnet, SSH
  • UDP: Connectionless, unreliable (best-effort), faster, lower overhead. Use for: DNS, SNMP, TFTP, streaming, gaming, VoIP

LAN vs WAN

  • LAN (Local Area Network): Covers small area (office, building). Owned by organization.
  • WAN (Wide Area Network): Covers large geographic area. Connection typically rented from service provider. Connects various sites/offices.

Switching — Quick Comparison

→Circuit switching: 3 phases (setup → transfer → disconnect), dedicated path, fixed data rate, voice networks
→Packet switching: store-and-forward, dynamic bandwidth, variable delay, data networks
→Datagram: independent packets, any route, may arrive out-of-order, receiver reorders
→Virtual circuit: pre-established path, VCI instead of destination address, ordered delivery
→Circuit switching: bandwidth guaranteed but wasted if idle
→Packet switching: efficient bandwidth use but possible packet loss/delay
