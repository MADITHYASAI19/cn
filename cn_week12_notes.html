Topic 1: Physical Layer — Foundations Exam Favorite

Role of the Physical Layer (Layer 1)

The physical layer is the lowest layer — it deals with the actual transmission of raw bits over a physical medium (copper wire, fiber optic, wireless). It converts digital data (0s and 1s) to physical signals (electrical voltages, light pulses, radio waves) and back. Everything above depends on the physical layer working correctly.

Key Physical Layer Concepts

1Bandwidth: The range of frequencies a medium can carry. Wider bandwidth → more data per second. Measured in Hz for analog; bps for digital.
2Bit Rate vs Baud Rate: Bit rate = number of BITS per second (bps). Baud rate = number of SIGNAL CHANGES per second (symbols/sec). If each symbol carries N bits, Bit rate = Baud rate × N. Example: QAM-16 → 4 bits/symbol → Bit rate = 4 × Baud rate.
3Nyquist Theorem: For a noiseless channel, max data rate = 2 × Bandwidth × log₂(M), where M = number of signal levels. If BW = 3000 Hz and M = 8: Max rate = 2 × 3000 × 3 = 18,000 bps.
4Shannon's Capacity: For a noisy channel, C = B × log₂(1 + S/N), where B = bandwidth, S/N = Signal-to-Noise ratio. This is the THEORETICAL MAXIMUM — you cannot exceed it regardless of encoding. Also: SNR (dB) = 10 × log₁₀(S/N).
5Transmission Media: Guided (wired) — twisted pair (UTP/STP), coaxial cable, fiber optic. Unguided (wireless) — radio waves, microwaves, infrared.

Topic 2: Digital Encoding Schemes Very High Weightage

Why Different Encoding Schemes?

Different line coding schemes have different properties regarding DC component (baseline wander), synchronization (clock recovery), bandwidth efficiency, and noise immunity. Choosing the right encoding is critical for reliable communication.

Non-Return-to-Zero (NRZ) Encoding

NRZ-L and NRZ-I

  • NRZ-L (Level): 1 → high voltage, 0 → low voltage. Constant level for duration of bit. Simple but has DC component and synchronization problems with long runs of same bit.
  • NRZ-I (Inverted): 1 → transition at start of bit; 0 → no transition. Better for 1s synchronization. Still has problem with long runs of 0s.
  • Main problems: No inherent clock signal. Long sequences of same bits cause baseline wander (DC drift). Receiver may lose synchronization.

Return-to-Zero (RZ) Encoding

RZ

  • Signal returns to ZERO in the MIDDLE of each bit period.
  • 1 → positive pulse then zero; 0 → negative pulse then zero.
  • Self-synchronizing (the mid-bit transition enables clock recovery).
  • Problem: Requires TWICE the bandwidth of NRZ — each bit has two signal changes.

Manchester Encoding (IEEE 802.3 Ethernet)

Manchester & Differential Manchester

  • Manchester (Biphase-L): Mid-bit transition ALWAYS occurs. 1 → low-to-high transition; 0 → high-to-low transition. Self-synchronizing. Used in original 10 Mbps Ethernet (10BASE-T).
  • Differential Manchester (Biphase-M): Transition at MIDDLE of every bit (for clocking). 1 → NO transition at START of bit; 0 → transition at START of bit. Used in Token Ring (802.5).
  • Advantages: No DC component; self-synchronizing; good noise immunity.
  • Disadvantage: Requires 2× bandwidth of NRZ (same as RZ) — each bit needs at least one transition.

4B/5B Encoding

4B/5B (used with NRZI)

  • Every 4 data bits are encoded as 5 output bits — a 5-bit code chosen to ensure enough transitions for clock recovery.
  • The 5-bit codes are selected so that no code has more than 3 consecutive 0s, preventing long runs of 0s.
  • 20% overhead (5 bits for 4 data bits). More bandwidth-efficient than Manchester (which has 100% overhead).
  • Used in Fast Ethernet (100BASE-TX with MLT-3 line coding), FDDI.
  • The 32 valid 5-bit patterns = 16 data codes + 16 control/special codes (idle, start, end-of-frame).

8B/10B and Other Modern Schemes

  • 8B/10B: 8 data bits → 10 channel bits. Used in Gigabit Ethernet (1000BASE-X), Fibre Channel, USB 3.0. Guarantees DC balance (equal 0s and 1s) and transition density.
  • 64B/66B: 64 data bits → 66 bits. Very low overhead (3%). Used in 10 Gbps and higher Ethernet.
  • PAM-4 (Pulse Amplitude Modulation-4): 4 voltage levels per symbol → 2 bits per symbol. Used in 25G, 50G, 100G Ethernet to double throughput without doubling baud rate.

Topic 3: Analog Modulation Techniques Exam Favorite

Why Modulation?

Digital data must often be converted to analog form for transmission over analog channels (e.g., telephone lines, wireless). Modulation is the process of varying a carrier wave's properties to carry information. A carrier is a continuous sine wave: A sin(2πft + φ).

Three Basic Analog Modulation Types

1. Amplitude Modulation (AM) — ASK

  • ASK (Amplitude Shift Keying): Digital version of AM. Different bit values → different amplitudes of carrier.
  • Binary ASK (OOK - On-Off Keying): 1 = carrier present; 0 = no carrier.
  • Pros: Simple to implement.
  • Cons: Very susceptible to noise and interference (amplitude is easy to corrupt). Poor noise immunity.
  • Used in: Fiber optic communication (optical on/off keying), some RFID.

2. Frequency Modulation (FM) — FSK

  • FSK (Frequency Shift Keying): Different bit values → different frequencies of carrier. 1 → higher frequency f₁; 0 → lower frequency f₀.
  • BFSK (Binary FSK): Two frequencies used. Common in modems (Bell 103).
  • Pros: Better noise immunity than ASK. Constant amplitude → less affected by amplitude noise.
  • Cons: Uses more bandwidth than ASK (needs two separate frequency bands).
  • Used in: Telephone modems (early), FM radio, some wireless sensors.

3. Phase Modulation (PM) — PSK

  • PSK (Phase Shift Keying): Different bit values → different phases of carrier. 1 → 0° phase; 0 → 180° phase (BPSK).
  • BPSK: Binary PSK — 1 bit/symbol (2 phases: 0° and 180°).
  • QPSK (Quadrature PSK): 4 phases (0°, 90°, 180°, 270°) → 2 bits/symbol. Doubles efficiency of BPSK in same bandwidth.
  • 8-PSK: 8 phases → 3 bits/symbol.
  • Pros: Better noise immunity than ASK; better bandwidth efficiency than FSK.

QAM — Quadrature Amplitude Modulation

  • Combines both amplitude AND phase variation. Most efficient modulation scheme.
  • QAM-16: 16 combinations (4 amplitudes × 4 phases) → 4 bits/symbol.
  • QAM-64: 64 combinations → 6 bits/symbol. Used in cable modems (DOCSIS), Wi-Fi.
  • QAM-256: 8 bits/symbol. Used in LTE, 802.11ac.
  • QAM-1024: 10 bits/symbol. Used in DOCSIS 3.1, 802.11ax (Wi-Fi 6).
  • Higher QAM → more bps BUT requires better signal quality (higher SNR) → shorter range in wireless.

Topic 4: Multiplexing — Sharing Transmission Media Exam Favorite

What is Multiplexing?

Multiplexing allows multiple signals (from different users/sources) to share a single transmission medium simultaneously. The device that combines signals is called a MULTIPLEXER (MUX). At the receiver, a DEMULTIPLEXER (DEMUX) separates them back. Key principle: efficient utilization of expensive high-bandwidth links.

Type 1: FDM — Frequency Division Multiplexing

FDM

  • Divide the total bandwidth into non-overlapping FREQUENCY BANDS.
  • Each user/signal is allocated a different frequency sub-band (channel).
  • Guard bands between channels prevent interference.
  • All users transmit simultaneously but on different frequencies.
  • Used in: Traditional AM/FM radio, cable TV (CATV), ADSL (telephone modem), OFDM in Wi-Fi and LTE.
  • Analog signals — no need for synchronization between users.
  • Problem: Wasted bandwidth if a user is inactive (their frequency band goes unused).

Type 2: TDM — Time Division Multiplexing

TDM — Synchronous (STDM) and Statistical

  • Divide time into SLOTS. Each user gets a fixed, recurring time slot.
  • Synchronous TDM (STDM): Each user is allocated a time slot in a repeating FRAME — whether they have data or not. Wasteful if users are idle → empty slots.
  • Statistical TDM (STDM / Stat-Mux): Time slots dynamically allocated to users based on DEMAND. Active users get slots; idle users get nothing. Much more efficient but requires headers to identify which slot belongs to which user.
  • T-carrier system (T1): 24 DS0 channels × 64kbps = 1.544 Mbps. Each DS0 = 8 bits × 8000 frames/sec.
  • E-carrier system (E1): European standard — 32 channels (30 data + 2 signaling) × 64kbps = 2.048 Mbps.

Type 3: WDM — Wavelength Division Multiplexing

WDM — Optical Multiplexing

  • FDM applied to OPTICAL FIBER. Different wavelengths (colors) of light carry different signals simultaneously on the same fiber.
  • DWDM (Dense WDM): 40, 80, or 160 channels on a single fiber, each at 10–100 Gbps. Total fiber capacity can reach multiple Tbps.
  • Wavelength range: approximately 1530–1565 nm (C-band) for DWDM.
  • Used in backbone fiber optic networks worldwide. The foundation of the modern internet infrastructure.

Type 4: CDM — Code Division Multiplexing

CDMA — Code Division Multiple Access

  • Each user is assigned a UNIQUE orthogonal code (spreading code). All users transmit simultaneously on the SAME frequency band at the SAME time.
  • Receiver uses each user's code to extract their signal from the composite (cross-correlation extracts one user's data).
  • Requires precise power control — if one user's signal is too strong, it overwhelms others (near-far problem).
  • Used in: 3G cellular networks (WCDMA, CDMA2000), GPS.

CIA Triad — The Three Goals of Security

  • Confidentiality: Only authorized parties can READ the information. Achieved via encryption (symmetric: AES, DES; asymmetric: RSA, ECC).
  • Integrity: Data has not been modified (tampered) in transit. Achieved via hash functions (MD5, SHA-256), digital signatures, MAC (Message Authentication Code).
  • Availability: Authorized users can ACCESS services when needed. Threatened by DoS/DDoS attacks, hardware failures.

Types of Attacks

Passive Attacks — Eavesdropping

  • Attacker only observes traffic — does NOT modify it. Much harder to detect.
  • Wire Tapping: Physically connect to a cable and read traffic.
  • Traffic Analysis: Even if encrypted, patterns (who talks to whom, how often, packet sizes) reveal information. Examples: Wireshark, packet sniffers.
  • Defense: Encryption (hides content), traffic padding (hides patterns).

Active Attacks — Modification

  • Attacker actively modifies or injects traffic. Easier to detect than passive.
  • Masquerade (Spoofing): Attacker pretends to be someone else (IP spoofing, MAC spoofing).
  • Replay Attack: Capture a valid message and retransmit it later to fool the receiver.
  • Modification: Intercept and alter messages in transit (man-in-the-middle).
  • Denial of Service (DoS/DDoS): Flood targets with traffic to exhaust resources and deny service to legitimate users.

Protective Mechanisms

Firewalls

  • A firewall is a security device (hardware or software) that controls traffic flow between networks based on rules.
  • Packet Filter Firewall: Inspects each packet individually based on source IP, destination IP, port, and protocol. Stateless — does not track connection state. Fast but limited.
  • Stateful Inspection Firewall: Tracks the STATE of connections. Only allows packets that are part of established connections. More secure than pure packet filtering.
  • Application-Layer Firewall (Proxy): Inspects content at the application layer. Can understand HTTP, FTP, DNS — most powerful but slowest.
  • DMZ (Demilitarized Zone): Network segment between external firewall (facing internet) and internal firewall (protecting private network). Public servers (web, email, DNS) placed in DMZ.

IPSec — Security at Network Layer

  • IPSec is a suite of protocols that provides security at the IP layer (Layer 3).
  • AH (Authentication Header): Provides authentication and integrity (but NOT confidentiality — no encryption of data).
  • ESP (Encapsulating Security Payload): Provides encryption (confidentiality), authentication, and integrity.
  • Two modes:
    • Transport mode: Only payload is encrypted/authenticated. Used for host-to-host communication.
    • Tunnel mode: Entire IP packet is encrypted/authenticated (new IP header added). Used for VPNs.
  • IKE (Internet Key Exchange) is used for key management and SA (Security Association) negotiation.

TLS/SSL — Security at Transport Layer

  • TLS (Transport Layer Security) provides encryption and authentication above TCP (effectively between Transport and Application layers).
  • HTTPS = HTTP + TLS. Port 443.
  • TLS Handshake: Client Hello → Server Hello + Certificate → Key Exchange → Finished. After handshake, all data is encrypted using symmetric keys derived during handshake.
  • Digital Certificates (X.509): Issued by Certificate Authorities (CAs). Bind a public key to an identity.

Encoding Scheme Summary

→NRZ-L: Simple, DC component, no sync. Use: RS-232, serial ports.
→Manchester: mid-bit transition, no DC, 2× BW, self-sync. Use: 10BASE-T Ethernet.
→Diff. Manchester: mid-bit = clock, start transition = 0/no-transition = 1. Use: Token Ring.
→4B/5B: 25% overhead, used with NRZI. Use: 100BASE-TX, FDDI.
→8B/10B: balanced, DC-free, 25% overhead. Use: GbE, USB 3.0, Fibre Channel.

Modulation Quick Facts

→ASK: amplitude changes; poor noise immunity; OOK = fiber optic on/off.
→FSK: frequency changes; better noise immunity; old modems.
→PSK: phase changes; BPSK=1bit, QPSK=2bits, 8PSK=3bits.
→QAM: amplitude + phase; QAM-16=4bits, QAM-64=6bits, QAM-256=8bits.
→More QAM levels → more bits/symbol BUT needs higher SNR.

Multiplexing Summary

→FDM: frequency bands; simultaneous; radio/TV/ADSL; guard bands prevent interference.
→TDM (Sync): fixed time slots; wastes slots when idle; T1=24ch=1.544Mbps; E1=32ch=2.048Mbps.
→Statistical TDM: dynamic slots; efficient; needs headers; used in packet networks.
→WDM/DWDM: multiple wavelengths on single fiber; Tbps capacity; backbone networks.
→CDMA: unique codes; all transmit same time/freq; 3G cellular; power control critical.
